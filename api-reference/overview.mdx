The Quorent REST API is a FastAPI-based HTTP API running at version 1.0.0. All endpoints are served under the `/api/` path prefix and communicate exclusively in JSON — both request bodies and response payloads. Authentication via a Supabase Bearer token is required on every endpoint except the internal cron routes.

## Base URL

When running locally, the server listens on port `8000` by default:

```bash
http://localhost:8000
```

All API endpoints are prefixed with `/api/`:

```bash
http://localhost:8000/api/
```

<Note>
  In production, replace `http://localhost:8000` with the public URL of your deployed backend.
</Note>

## Request and response format

Every request that includes a body must send `Content-Type: application/json`. Every response from the API is also JSON. There is no XML or form-encoded support.

```json
{
  "Content-Type": "application/json",
  "Authorization": "Bearer YOUR_TOKEN"
}
```

## Authentication

All protected routes require a valid Supabase JWT passed as a Bearer token in the `Authorization` header. Requests without a valid token return a `401 Unauthorized` response. See the [authentication guide](/api-reference/authentication) for details on obtaining and using your token.

<Info>
  The `/api/cron` routes use a separate `cron_secret_token` for internal scheduled jobs and are not intended for direct API consumers.
</Info>

## API version

The current API version is **1.0.0**.
